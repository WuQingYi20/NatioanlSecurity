 Project Sentinel-KG: Guardrailed AI Agents for National Security Knowledge Graph Traversal📋 Executive SummaryProject Sentinel-KG proposes an autonomous, multi-agent reasoning system designed for the Swedish government (integrating data from the Tax Agency, Social Insurance Agency, and Bolagsverket). Inspired by advanced Cyber Reasoning Systems (like DARPA AIxCC's Atlantis), this system deploys Large Language Model (LLM) Agents to traverse a massive national security Knowledge Graph (KG).To ensure strict compliance with Swedish privacy laws and GDPR, the agents are trained using a Reinforcement Learning (RL) pipeline with dynamic LoRA (Low-Rank Adaptation) fine-tuning, operating entirely within an on-premise, air-gapped environment.🎯 Core ObjectivesAutonomous Threat Hunting: Replace manual, linear database queries with heuristic, multi-hop graph traversal.Privacy by Design (Guardrails): Implement strict "Virtual File System" (VFS) style sandboxing to prevent unauthorized data access.Continual Alignment: Use Reinforcement Learning to continuously train agents to balance threat detection accuracy against civilian privacy protection.⚙️ The Reinforcement Learning Pipeline (Architecture)The core innovation of this system is modeling the threat-hunting process as a Reinforcement Learning problem. The agent learns to navigate the data maze not by rigid rules, but through trial, error, and strict algorithmic guardrails.Code snippetgraph TD

🧩 Pipeline Breakdown1. State (Observation)The agent does not see the entire national database. Its state is a semantic window: a localized subgraph containing the current entity under investigation, its immediate multi-hop connections (financial, social), and the history of the agent's previous queries.2. Action SpaceThe agent's actions are strictly defined by a protocol (similar to the P4 framework):explore_node(entity_id): Move to a connected node.request_clearance(level): Ask for human approval to access classified layers.submit_evidence_bundle(): Conclude the search and report a potential threat.3. Environment (The Guardrailed Sandbox)Agents operate in a read-only, simulated layer. They cannot alter government records or directly freeze assets. The environment evaluates the logic of the agent's traversal path.4. The Reward Function (The Core Guardrail)The RL reward function mathematically enforces Swedish ethical standards:$$Reward = (\alpha \times \text{Threat\_Confidence}) + (\beta \times \text{Efficiency}) - (\gamma \times \text{Privacy\_Invasion\_Penalty})$$If the agent digs too deeply into an innocent citizen's unrelated medical or tax records, the $\gamma$ penalty massively outweighs the score, teaching the agent to avoid this behavior.🔒 Key TechnologiesOn-Premise Local LLMs: To guarantee data sovereignty, the system uses locally hosted open-source models (e.g., Llama-3 or Mistral). No data ever leaves the government's intranet.LoRA (Low-Rank Adaptation): Instead of full-parameter fine-tuning, the system uses LoRA to quickly adapt the agent's behavior. When new privacy laws are passed, a small LoRA plugin is trained and hot-swapped into the model within hours.GRPO (Group Relative Policy Optimization): Multiple agents are deployed simultaneously on the same anomaly. They are scored relative to each other, rapidly optimizing the traversal strategy without requiring a complex critic model.🛑 Ethical & Operational GuardrailsAddressing the committee's operational and ethical concerns:Anti-Hallucination: Agents cannot fabricate connections. Every edge and node referenced in their final Evidence Bundle must cryptographically map back to a real database entry.Human-in-the-Loop (HITL): The system is strictly an advisor. It compiles the narrative and the evidence graph, but a human operator must make the final determination to act.Auditability: Because every Action and State transition is logged as a reinforcement learning trajectory, oversight committees can mathematically audit why the AI decided to investigate a specific individual.🚀 Next Steps[ ] Phase 1: Construct a synthetic (fake data) knowledge graph to simulate Swedish agency data without violating GDPR.[ ] Phase 2: Implement the Environment Sandbox and the Action API interface.[ ] Phase 3: Train the baseline model using imitation learning, followed by the RL-LoRA pipeline to enforce the privacy penalty ($\gamma$).
